    		Building an Interactive Website to Validate ChatGPT Efficiency

    Research approach

    This project utilizes MongoDB, Node.js, and the ChatGPT API to evaluate ChatGPT's efficiency across multiple domains. The website integrates client-server architecture to process user interactions, retrieve data, and present results dynamically. Node.js serves as the backend to manage API requests and responses, while MongoDB stores datasets and evaluation results for further analysis.

    Dataset Overview

    The dataset consists of diverse questions from three domains: History, Social Science, and Computer Security. Each question is designed to test ChatGPT’s understanding, reasoning, and ability to provide accurate answers. The evaluation aims to identify strengths and weaknesses in ChatGPT’s performance, providing insights into its reliability across these fields.

    Results

    This section includes visualizations generated from the evaluation results. Data charts and graphs present ChatGPT’s performance metrics, such as accuracy, response time, and domain-specific trends. The visualizations help to identify patterns and comparative strengths in each domain.
