Building an Interactive Website to Validate ChatGPT Efficiency

Research approach

This project utilizes MongoDB, Node.js, and the ChatGPT API to evaluate ChatGPT's efficiency across multiple domains. The website integrates client-server architecture to process user interactions, retrieve data, and present results dynamically. Node.js serves as the backend to manage API requests and responses, while MongoDB stores datasets and evaluation results for further analysis.

Dataset Overview

The dataset used for evaluating ChatGPT consists of questions from three domains: History, Social Science, and Computer Security. These areas were selected to test ChatGPT’s ability to answer questions across different topics, providing insights into its performance. The goal of the evaluation is to measure how well ChatGPT responds to questions in each area, while also identifying strengths and areas for improvement.

Findings:

  • The average time to process/save Social Science questions was 8895.96ms, which is longer than the times for History and Computer Security.
  • The fastest questions processed were from the History domain, with an average time of 6964.44ms.
  • All questions were answered with detailed explanations.
